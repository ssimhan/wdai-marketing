## Post-Session: The Feedback Loop

The session content doesn't end when the demos do. What presenters built, what the audience reacted to, any standout quotes — that's the raw material for follow-on content. But it only gets captured if someone writes it down right after.

**Carolyn's job after each session**: Drop the structured recap (see Step 7 in SKILL.md) into a designated Slack thread. Ideally the same day, while it's fresh.

**What happens next**: Sheena or Sandhya runs adhoc-content-activator on the recap to turn it into:
- An internal spotlight (#general or cohort channels)
- A LinkedIn post featuring a presenter's build
- A newsletter mention
- Social proof for the next cohort's AI Foundations promotion

The recap → adhoc-content-activator pipeline is how SDT output compounds over time. Without it, each session is a one-time event. With it, every session generates content that promotes the next one.

## Copy Tone for Show Don't Tell

- **Concrete and specific**: Name the presenter, describe what they built in plain language. "Anna built an AI cooking assistant that suggests recipes based on what's in your fridge" is infinitely more compelling than "a member demo on AI tools."
- **Peer-to-peer energy**: SDT is members showing other members. Keep the copy at that level — not a formal event announcement, more like "you're going to want to see what [Name] built."
- **Low friction**: The CTA should feel easy. "Come watch. Ask questions. Get inspired." Not "Register now before seats run out."
- **Authentic excitement**: If Carolyn's excited about what someone built, that should be in the copy. Genuine enthusiasm is more persuasive than polished promotion.

## What to Collect from Carolyn at the Start

When running this skill for an SDT session, ask Carolyn (or the person triggering the plan) for:

1. Session date (confirm against third Thursday)
2. Presenter names (all 3)
3. One-line description of each presenter's build (in plain language)
4. Any themes or connections across the three builds
5. Whether she wants a LinkedIn post for this one (her call)
6. Luma registration link (if applicable)

Without the presenter names and build descriptions, the promotion is generic. The whole point of SDT is specificity — real people, real builds.
